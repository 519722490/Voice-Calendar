# 前端 Bug 修复记录

## 2026-05-31 语音弹窗关闭后无法再次打开

### 问题现象

在手动确认语音模式下，点击“开始录音”后直接关闭语音弹窗，再次点击麦克风按钮时无法重新打开语音弹窗。

### 原因分析

关闭语音弹窗时已经清理了 WebSocket 和浏览器音频采集资源，但没有把 `voiceStatus` 从 `connecting` / `recording` 恢复为 `idle`。

顶部麦克风按钮依赖 `canUseVoiceEntry` 判断是否可用，当 `voiceStatus` 仍处于录音相关状态时，按钮会保持禁用。

### 修复方案

新增录音中断清理流程：

- 关闭语音弹窗时清除自动提交定时器。
- 关闭语音识别 WebSocket。
- 停止麦克风音频流和 AudioWorklet。
- 将 `voiceStatus` 恢复为 `idle`。
- 清空本次识别文本和错误提示，保证关闭弹窗后不触发 Agent，也不保留半截语音内容。

## 2026-05-31 审查模式缺少取消执行入口

### 问题现象

Agent 审查模式返回待确认操作后，用户只能点击“确认执行”，没有明确的“取消执行”按钮。

### 原因分析

前端只实现了 `/api/agent/confirm` 的确认流程。用户如果不想执行，只能关闭结果弹窗，这会清空前端状态，但后端待确认操作仍会等到过期时间后才失效。

### 修复方案

新增取消执行流程：

- 后端新增 `/api/agent/cancel` 接口。
- `AgentConfirmationStore` 增加 `cancelAny` 方法，支持取消普通日程和重复日程的待确认操作。
- 前端 Agent 结果弹窗中，所有待确认操作旁边都显示“取消执行”按钮。
- 点击取消后，前端会调用后端取消接口，并将对应待确认项标记为“已取消执行”。

### 验证方式

- 在录音状态关闭语音弹窗，再次点击麦克风按钮，应能正常打开。
- 在审查模式下提交会产生待确认操作的语音文本，结果弹窗应同时显示“取消执行”和“确认执行”。
- 点击“取消执行”后，该待确认操作应消失，不能再被确认执行。
